## 🎯 Overview

**IDS** is an enterprise-grade intrusion detection system that combines cutting-edge machine learning with real-time network analysis:

- 🔍 **Monitors** network traffic in real-time
- 🤖 **Detects** threats using ensemble ML models (99.1% accuracy)
- 📊 **Analyzes** patterns with Jupyter-based data science workflows
- 🔔 **Alerts** security teams with intelligent notifications
- 📈 **Visualizes** threats via interactive React dashboards
- 📋 **Generates** comprehensive security reports
- 🔐 **Protects** with enterprise-grade security features
- 🐳 **Deploys** with Docker for easy scaling

**Perfect for**: SOC teams, network security researchers, penetration testing labs, threat intelligence platforms

## 🚀 Quick Start

### Option 1: Docker (⭐ Recommended)

```bash
# Clone repository
git clone https://github.com/lalitsingh789/IDS-.git
cd IDS-

# Start all services
docker-compose up -d

# Wait for services to start (30 seconds)
sleep 30

# Access services
echo "Dashboard:     http://localhost:3000"
echo "Backend API:   http://localhost:5000"
echo "ML Server:     http://localhost:5001"
echo "PostgreSQL:    localhost:5432"
```

### Option 2: Manual Setup

#### Step 1: Install Prerequisites
```bash
# Check Python
python --version  # Should be 3.8+

# Check Node.js
node --version    # Should be v14+
npm --version     # Should be v6+

# Check Git
git --version
```

#### Step 2: Clone & Navigate
```bash
git clone https://github.com/lalitsingh789/IDS-.git
cd IDS-
```

#### Step 3: Setup Python Environment
```bash
# Create virtual environment
python -m venv venv

# Activate it
# On macOS/Linux:
source venv/bin/activate
# On Windows:
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

#### Step 4: Setup Backend
```bash
cd backend

# Install Node dependencies
npm install

# Setup environment
cp .env.example .env

# Edit .env file with your settings
nano .env

# Start backend server
npm run dev

# Server running on http://localhost:5000
```

#### Step 5: Setup Frontend
```bash
cd ../frontend

# Install dependencies
npm install

# Setup environment
cp .env.example .env

# Configure API URL in .env
# REACT_APP_API_URL=http://localhost:5000

# Start development server
npm start

# Dashboard running on http://localhost:3000
```

#### Step 6: Train ML Models
```bash
cd ../backend/ml-models

# Run training notebooks
jupyter notebook

# Or execute training script
python train_models.py

# Verify models were saved
ls -la models/trained_models/
```

## 📡 API Reference

### Authentication

```bash
# Login & Get JWT Token
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "admin123"
  }'

# Response:
# {
#   "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
#   "user": {
#     "id": "1",
#     "email": "admin@example.com",
#     "role": "admin"
#   }
# }

# Use token in subsequent requests:
# Authorization: Bearer YOUR_TOKEN_HERE
```

### Get Alerts

```bash
# Get all alerts
curl -X GET http://localhost:5000/api/alerts \
  -H "Authorization: Bearer YOUR_TOKEN"

# Get single alert
curl -X GET http://localhost:5000/api/alerts/123 \
  -H "Authorization: Bearer YOUR_TOKEN"

# Filter by severity
curl -X GET "http://localhost:5000/api/alerts?severity=high" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### ML Predictions

```bash
# Get threat prediction
curl -X POST http://localhost:5000/api/predict \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "features": [0.5, 0.3, 0.8, 0.2, ...],
    "model": "ensemble"
  }'

# Response:
# {
#   "prediction": 1,
#   "probability": 0.991,
#   "severity": "high",
#   "confidence": 0.989,
#   "timestamp": "2026-05-15T10:30:00Z"
# }
```
